Ebie & Eleanor

HTML CSS JavaScript
[ Live   /   GitHub Repo ]

Ebie & Eleanor is a small jewelry business operating out of Greenville, South Carolina. They make bracelets out of 100% recycled, repurposed materials like leather, metal wire, etc. I love what they’re doing, but I noticed they didn’t have much of a web presence. I reached out and next thing you know, they have a beautiful web brochure showing off their work! It’s built with HTML, SASS, and JavaScript. Last.fm Music Search

HTML CSS JavaScript
[ Live   /   GitHub Repo]

I love music. Before I decided to be a web developer I was a musician for most of my life. While thinking of ideas for apps to make for practice I decided to blend my love for music with my love for programming, and the result was a react app that leverages the Last.fm api to allow users to look up artists or bands and get pictures, bios, pictures of the artist/band’s most popular albums as well as the play count for those albums.

Building the skeletons of web pages using semantic tags and container based structures, my HTML skills are a force to be reckoned with. Well, they’re pretty good at least. HTML is not exactly rocket science but I’ve learned most of what there is to learn about it. SEO, accessibility, meta tags, you name it, I know what the deal is.

Writing out stylesheets is one of my favorite parts of web development. I’m not exactly a designer but I like to think I have an eye for it. Recently I started using preprocessors, namely SASS, to really get serious about my stylesheets. I like to keep my styles coordinated and formatted in an easy to read, easy to maintain manner. I’ve been trying my best to use the BEM model as well.

I’ve been learning JavaScript for almost 2 years now. From DOM manipulation to complex application logic, I’ve developed a very strong set of fundamental JS skills. I have a good understanding of OOP and functional programming paradigms (my favorite being functional!) and employ them whenever I can in my applications. JavaScript has a lot of gotchas like the this keyword and I’ve done my best to have knowledge of these things and how to work with and around them.

After solidifying my JavaScript knowledge I set my sights on learning a framework. React seemed to be the most interesting and after several months of studying and practice, I can confirm that it is indeed awesome and cool and good. I’ve learned to break down my applications into manageable components. I employ best practices such as keeping state immutable, delegating helper functions to their own file, using closures, using functional components where state is not necessary, etc.
